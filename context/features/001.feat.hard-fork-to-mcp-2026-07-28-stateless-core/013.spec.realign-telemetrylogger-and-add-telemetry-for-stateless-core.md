# Realign TelemetryLogger and add telemetry for stateless core

`GenMCP.TelemetryLogger` was never updated for the stateless hard fork. It still
handles events from the deleted session/cluster machinery, and the one new event
the stateless core actually emits is unhandled. We realign it to what the new
core emits and add telemetry at the interesting points of the request flow.

## Current mismatch (verified 2026-06-22)

Events **emitted** in the current lib (`server.ex` only):

- `[:gen_mcp, :server, :init]` (`server.ex:88`) — handled ✓
- `[:gen_mcp, :server, :start_error]` (`server.ex:57`) — **not handled** ✗
  (in `:test` env the `handle_event` catchall halts the VM, so this is latent).

Events **handled** by `TelemetryLogger` but no longer emitted anywhere:

- `[:gen_mcp, :cluster, :conflict]`, `[:gen_mcp, :cluster, :error]` —
  `syn_event_handler.ex` was deleted; there is no clustering in the stateless core.
- `[:gen_mcp, :suite, :error, :unknown_request]` — `suite.ex` no longer emits it.

For reference, `main` emitted session lifecycle (`init` / `restore` /
`restore_error` / `delete` / `terminate` / `start_error`) from `mux/session.ex`,
cluster `conflict` from `syn_event_handler.ex`, and the suite `unknown_request`.
All of that source machinery is gone in the fork.

## Event contract (finalized)

Levels: server crashes are `:error`; client/request rejections are `:debug`.
Version negotiation gets its **own** event so its level can move independently.

| Event | Level | Metadata | Trigger |
|---|---|---|---|
| `[:gen_mcp, :server, :init]` | `:debug` | `%{server_mod, server_arg, owner}` | worker boot (existing emit, `server.ex:88`) |
| `[:gen_mcp, :server, :start_error]` | `:error` | `%{reason}` | worker failed to start (existing emit, `server.ex:57`; add handler) |
| `[:gen_mcp, :transport, :server_crashed]` | `:error` | `%{reason}` | worker died before replying (`:server_crashed` / `handle_server_down`) |
| `[:gen_mcp, :transport, :version_rejected]` | `:debug` | `%{reason}` | `{:unsupported_protocol_version, _}` (own event, tunable) |
| `[:gen_mcp, :transport, :request_rejected]` | `:debug` | `%{reason}` | every other rejection (origin, bad_rpc(_version), header mismatch/missing, invalid params/level, unknown method/tool, …) |

## Plan

1. **Drop the dead handlers** for `:cluster` events and the old
   `:suite, :error, :unknown_request` mapping (no emitters in the stateless core).
2. **Handle `[:gen_mcp, :server, :start_error]`** — already emitted, unhandled.
3. **Emit the transport events from one funnel.** All rejections already converge
   on `send_error/3` (→ `GenMCP.Error.cast_error/1`), so the natural single
   emission point classifies the `reason` there:
   - `:server_crashed` → `:transport, :server_crashed`
   - `{:unsupported_protocol_version, _}` → `:transport, :version_rejected`
     (unique tag → clean split; the protocol-version *header* mismatch stays in the
     generic bucket)
   - anything else → `:transport, :request_rejected`

Keep the `:gen_mcp` event namespace and the existing `filter_events/2` design
(min level + prefixes). Per project convention, **no `@doc`/`@moduledoc`** added
to new/changed code in this fork yet (doc rewrite is separate). The `@moduledoc`
table in `TelemetryLogger` is generated from the `events` map, so it stays
in sync automatically.

## Work split

Implemented in one session: spec + `TelemetryLogger` isolation test
(`test/gen_mcp/telemetry_logger_test.exs`) and the production code — the
`send_error/3` funnel classification in `transport/streamable_http.ex` and the
realigned `TelemetryLogger` handlers/`events` map.

## Verification

- The `:test`-env `handle_event` catchall (halts the VM on an unhandled event) is
  the safety net: every emitted event must have a matching handler or the suite
  dies. Run `mix test`.
- A focused `TelemetryLogger` test attaches the listener, emits each event, and
  asserts the expected log line / level (capture log).
- `mix check` green.

## Acceptance criteria

- No handler references an event the lib no longer emits; no emitted event is
  unhandled (catchall never fires in tests).
- `[:gen_mcp, :server, :start_error]` produces an `:error` log.
- New telemetry points added at the agreed request-flow sites, each mapped to a
  log level.
- `mix check` green.
