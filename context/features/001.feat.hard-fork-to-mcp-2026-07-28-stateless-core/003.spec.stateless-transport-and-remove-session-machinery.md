# Stateless transport + remove the session machinery

The core of the fork. Make `StreamableHTTP` dispatch statelessly and **delete**
the session spine. This is the big simplification.

## Delete (no longer needed)

- `lib/gen_mcp/mux/session.ex` -- the per-session GenServer.
- `lib/gen_mcp/mux/session_supervisor.ex` -- the DynamicSupervisor.
- `lib/gen_mcp/suite/session_controller.ex` + `session_controller/` (Noop,
  DevSessionStore) -- persistence/restore.
- `lib/gen_mcp/suite/persisted_client_info.ex` (if only used for restore).
- `syn` usage + the `:gen_mcp_sessions` scope; drop the `:syn` dep from
  `mix.exs` and `GenMCP.Application` start tree if unused elsewhere.
- `GenMCP.Mux` session functions (`start_session`, `ensure_started`,
  `delete_session`, `lookup_pid`, ...). `Mux` may shrink to nothing or be
  removed; keep only what the stateless path needs (likely just `Channel`).

## Rework `lib/gen_mcp/transport/streamable_http.ex` (Impl)

- **Remove** `Mcp-Session-Id` entirely: `@session_id_header`,
  `lookup_session_id/1`, `reject_session_id/1`, `put_resp_session_id/2`,
  `terminate_session/2`, the `delete "/"` route's session deletion. The HTTP
  `DELETE` either 405s or becomes a no-op per the RC.
- **Version**: read the authoritative `MCP-Protocol-Version` HTTP header
  (SEP-1442); reconcile with `_meta` `io.modelcontextprotocol/protocolVersion`
  against the pinned draft. Single supported version -> reject mismatches with a
  clear error (no negotiation table).
- **Routing headers**: parse `Mcp-Method` / `Mcp-Name` (validate-but-tolerate).
- **Dispatch**: replace `Mux.start_session` / `ensure_started` /
  `Mux.request` with a direct call into the stateless runner (spec 004),
  building an ephemeral `Channel` (`Channel.from_request/2` already targets
  `self()` -- no registry). SSE streaming, keepalive, and the `stream_loop/1`
  machinery (lines 344-477) are **kept** -- they are transport concerns, not
  session concerns.
- Extract `_meta` `io.modelcontextprotocol/*` fields once per request into the
  channel `assigns`.

## `GenMCP.Mux.Channel`

Largely unchanged -- it is the reply path, not session state. Confirm
`from_request/2` (line 26) reads `progressToken` from the 2026 `_meta` shape
(the token may sit beside the new reverse-DNS keys).

## Tests

- Rewrite the conn-case suite for stateless: `ping`, `tools/list`,
  `tools/call`, `resources/*`, `prompts/*`, `server/discover` round-trip with
  **no** session header and **no** `mcp-session-id` in responses.
- Delete the session/restore/timeout tests (`test/gen_mcp/mux/...`).
- Load-balancer invariant: two sequential requests with no shared state succeed
  independently.

## Acceptance criteria

- No session GenServer/registry exists; `syn` removed if unused.
- Stateless requests round-trip; SSE streaming still works.
- `mix check` green; dialyzer clean after the deletions.
