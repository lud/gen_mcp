# Foundation -- schema vendoring + versioned codegen

Get the 2026 vocabulary generated and the protocol version centralized. No
transport/dispatch changes yet -- this phase just produces the new entity
structs and the constant that names the version.

## Vendor the draft schema (`mix.exs`)

`mcp_schemas/0` (lines 71-83) sparse-checkouts `schema/#{@schemas_vsn}` pinned to
the `2025-11-25` ref. Repoint it at **`schema/draft`** and pin to a **commit
SHA** (not a moving ref) since `draft/` changes as SEP PRs land. Keep
`only: [:dev, :test], compile: false, runtime: false`. Re-run codegen +
`check-entities` whenever the SHA is bumped.

Source URL for reference:
`https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/schema/draft/schema.json`
(metaschema JSON Schema 2020-12).

## Versioned codegen (`tools/gen-rpc-schemas.exs`)

- `base_module/0` (line 737) returns `GenMCP.MCP` and feeds `module_name/1`,
  `prelude/0` (lines 393-432), `mod_map/2`. Parameterize it to emit
  **`GenMCP.MCP.V2607`** so structs land at `GenMCP.MCP.V2607.Tool`, etc., and
  the builder/`ModMap`/`Meta` preludes match. Output to
  `lib/gen_mcp/mcp/v2607/entities.ex` (update `modkit` mount + `docs` grouping).
- `module_config/1` (lines 146-302) -- rebuild the allow/deny table from the
  **draft** `$defs` (do not assume parity with `2025-11-25`):
  - **Generate** the new core types: `DiscoverRequest` / `DiscoverResult`,
    `MetaObject` / `RequestMetaObject` (the `_meta` carriers, reverse-DNS keys),
    `InputRequiredResult`, `InputRequest(s)`, `InputResponse(s)`,
    `InputResponseRequestParams`, `SubscriptionsListen*`.
  - **Absent from core**: `initialize` handshake structs (`InitializeRequest` /
    `InitializeResult` / `InitializedNotification`) and Task* types. Mark
    `:nogen` if present as legacy; Tasks come from the extension (spec 005).
  - **Cache hints** `ttlMs` / `cacheScope`: verify whether the draft actually
    carries these field names on list/read results before relying on them
    (not confirmed during research) -- generate if present.

## Version constant (`lib/gen_mcp.ex`)

`supported_protocol_versions/0` (lines 73-75) currently lists the two 2025
versions. For the fork it becomes single-version:

```elixir
def protocol_version, do: "2026-07-28"  # the draft we target until finalization
```

Drop the multi-version list and the `2025-*` entries. `intialize_result/1` in
the builder no longer applies (no handshake) -- the version surfaces via
`DiscoverResult` / `_meta` instead (wired in specs 003-004).

## `GenMCP.MCP` facade

Keep `GenMCP.MCP` as a thin module re-exporting / delegating to
`GenMCP.MCP.V2607` (builder helpers + struct aliases) so user code and internal
`alias GenMCP.MCP` sites keep working. The builder module
(`lib/gen_mcp/mcp.ex`) is regenerated/relocated under `V2607`; the facade points
at it.

## Config sweeps (`mix.exs`)

- `modkit` mount (line ~84) -- add `GenMCP.MCP.V2607` mapping.
- `docs`: `nest_modules_by_prefix` / `groups_for_modules` (lines ~136-175) --
  update for the versioned namespace, keep hexdocs grouping coherent.
- `test_coverage` `ignore_modules` `~r{^GenMCP\.MCP\.}` -- confirm coverage.
- `tools/check-entities.exs` -- point at the `V2607` namespace.

## Acceptance criteria

- `GenMCP.MCP.V2607.*` entities compile from the SHA-pinned draft schema.
- `GenMCP.MCP` facade resolves; `check-entities` green.
- `mix check` green. No dispatch/transport behaviour changed yet.
