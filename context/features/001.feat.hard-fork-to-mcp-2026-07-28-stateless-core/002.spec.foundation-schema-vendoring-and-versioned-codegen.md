# Foundation -- schema vendoring + versioned codegen

Get the 2026 vocabulary generated and the protocol version centralized. No
transport/dispatch changes yet -- this phase just produces the new entity
structs and the constant that names the version.

## Vendor the draft schema (`mix.exs`)

`mcp_schemas/0` (mix.exs ~78-87, with `@schemas_vsn` at ~76) sparse-checkouts
`schema/#{@schemas_vsn}` pinned to the `2025-11-25` ref. Repoint it at
**`schema/draft`** and pin to a **commit SHA** (not a moving ref) since `draft/`
changes as SEP PRs land. Keep
`only: [:dev, :test], compile: false, runtime: false`. Re-run codegen +
`check-entities` whenever the SHA is bumped.

Source URL for reference:
`https://raw.githubusercontent.com/modelcontextprotocol/modelcontextprotocol/main/schema/draft/schema.json`
(metaschema JSON Schema 2020-12).

## Versioned codegen (`tools/gen-rpc-schemas.exs`)

**The generator is now fully parameterized** (refactor landed in `9cd98cf` /
`a6f43a1`). `Generator.run/2` takes the source schema path plus `output_path:`,
`mod_prefix:`, and `mod_config:`; a `Context` carries `mod_prefix`/`mod_config`
and `Codegen.module_name(name, ctx)` derives module names from the prefix
(`prelude/1` and `mod_map/3` pick it up automatically). The old hardcoded
`base_module/0` / `module_config/1` are gone -- the parameterization work this
section originally described is **already done**.

What remains is to drive the generator for the draft. At the bottom of the file
the call currently targets `2025-11-25` / `GenMCP.MCP`; point it at the draft:

- `source_schema_path` -> `deps/modelcontextprotocol/schema/draft/schema.json`
  (the SHA-pinned dep added below).
- `mod_prefix: GenMCP.MCP.V2607` so structs land at `GenMCP.MCP.V2607.Tool`,
  etc.
- `output_path: "lib/gen_mcp/mcp/v2607/entities.ex"` (update `modkit` mount +
  `docs` grouping below).
- **Rebuild the `mod_config` allow/deny table from the draft `$defs`** -- do not
  assume parity with `2025-11-25`. A `$defs`-vs-`mod_config` diff is the first
  build step (the generator raises on any `$def` missing from `mod_config`):
  - **Generate** the new core types: `DiscoverRequest` / `DiscoverResult`,
    `MetaObject` / `RequestMetaObject` (the `_meta` carriers, reverse-DNS keys),
    `InputRequiredResult`, `InputRequest(s)`, `InputResponse(s)`,
    `InputResponseRequestParams`, `SubscriptionsListen*`.
  - **Absent from core**: `initialize` handshake structs (`InitializeRequest` /
    `InitializeResult` / `InitializedNotification`) and Task* types. Mark
    `:nogen` if present as legacy; Tasks come from the extension (spec 005).
  - **Cache hints** `ttlMs` / `cacheScope`: verify whether the draft actually
    carries these field names on list/read results before relying on them
    (not confirmed during research) -- generate if present.

## Version constant (`lib/gen_mcp.ex`)

`supported_protocol_versions/0` (lines 73-75) currently lists the two 2025
versions. **Additive scope:** *add* the centralized constant now without
removing the old list (the old list still feeds the live `initialize` handshake
that 003-004 will rip out):

```elixir
def protocol_version, do: "2026-07-28"  # the draft we target until finalization
```

Dropping the multi-version list / `2025-*` entries and retiring
`intialize_result/1` (no handshake; version surfaces via `DiscoverResult` /
`_meta`) happens in **003-004** when dispatch is reworked. 002 only introduces
the constant.

## `GenMCP.MCP` facade (deferred to 003-005)

The end state is `GenMCP.MCP` as a thin facade delegating to `GenMCP.MCP.V2607`.
**This is NOT done in 002** under the additive decision: repointing
`GenMCP.MCP` at V2607 or relocating the builder would break the still-present
dispatch/session code that references initialize-era structs absent from V2607
(`InitializeRequest`, `InitializeResult`, `InitializedNotification`, ...).

In 002, `GenMCP.MCP` (builder, `lib/gen_mcp/mcp.ex`) and the old
`GenMCP.MCP.*` entities stay exactly as they are; `GenMCP.MCP.V2607.*` is
generated alongside them. The facade/relocation lands in 003-005 when the spine
is amputated and the old vocabulary is deleted.
